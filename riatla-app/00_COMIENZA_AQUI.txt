🎯 PROCESO COMPRENDIDO
════════════════════════════════════════════════════════════════════════════

Riatla es un sistema de avatar VRM interactivo:

1. Home Assistant publica eventos MQTT
   └─ Topic: riatla/emocion, riatla/reset, riatla/estado

2. riatla_daemon.py escucha MQTT
   └─ Recibe comandos y genera WebSocket

3. riatla-app es el cliente Electron
   ├─ Carga modelo VRM (riatla.vrm)
   ├─ Renderiza con Three.js
   ├─ Escucha WebSocket
   └─ Anima avatar en respuesta a comandos

4. Expresiones y movimientos se aplican en tiempo real
   └─ BlendShapes para expresiones faciales
   └─ Bone rotations para movimientos

🎮 COMANDOS SOPORTADOS
════════════════════════════════════════════════════════════════════════════

WebSocket JSON que envía riatla_daemon:

  {"accion": "emocion_happy"}      → Avatar sonríe
  {"accion": "emocion_sad"}        → Avatar triste
  {"accion": "emocion_angry"}      → Avatar enojado
  {"accion": "emocion_surprised"}  → Avatar sorprendido
  {"accion": "emocion_relaxed"}    → Avatar relajado
  {"accion": "emocion_neutral"}    → Avatar neutral
  
  {"accion": "mirar", "parametros": {"x": 0.3, "y": 0.1, "z": 0}}
  → Rotación de cabeza (ejes x, y, z)

  {"accion": "reset"}              → Vuelve a neutral
